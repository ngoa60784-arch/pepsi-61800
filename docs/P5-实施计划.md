## 子阶段与验收

### P5-A — FP-06 LocalProcessBackend（M）✅

**目标**：`runtime.solverHost=local` 时 `Bun.spawn` 本机 `tch-agent solver rpc`，JSONL 与现 Runtime 一致。

**涉及**

- `packages/core/src/runtime/execution-backend.ts` — 实现 `LocalProcessBackend.spawn/stop`
- `packages/core/src/runtime/runtime.ts` — `createExecutionBackend` 按配置切换
- `packages/ui-web/src/components/config/host.tsx` — Solver 宿主选项
- 回归：`solverHost=docker` 行为不变

**验收**：无 Docker 时可 launch Solver 并完成一轮工具调用。

---

### P5-B — FP-06 execSurface（M）✅

**目标**：`execSurface=local-host` 时 Prompt 允许本机 bash 扫打；`remote-vps` 保持现状。

**涉及**

- `packages/core/src/config/prompts/builtin/kimi-security.md` 等 — 按 `TCH_EXEC_SURFACE` 切换规则
- Web 配置页「命令执行位置」
- Engagement scope 文档提示

**验收**：`local-host` + 本机工具可用时对授权目标可执行命令；`remote-vps` 回归通过。

---

### P5-C — 单二进制 sidecar（M）✅

**目标**：`bun compile` 产出控制面单二进制，内嵌 runtime/mcp 资源路径约定。

**涉及**

- `scripts/build.ts` + `generate-builtin-assets.ts`（内嵌 MCP `.py`）
- `initBuiltinMcpScripts` / `TCH_MCP_DIR` / `resolveMcpConfigPathForSession`
- sidecar 启动时 `web` / `solver rpc` 均执行 `ensureHostStaticConfig`

**验收**：编译产物可 `web` / `solver rpc` 无源码树依赖。

---

### P5-D — Tauri 壳（L）🚧

**目标**：新建 `apps/desktop/`（Tauri 2），单实例、无地址栏 WebView、退出收尾。

**涉及**

- `apps/desktop/`、`tauri.conf.json` — **骨架已建**
- `scripts/prepare-desktop-sidecar.ts`、`scripts/pack-desktop.sh`
- spawn P5-C sidecar（发行）或 `bun` 开发回退，加载 `http://127.0.0.1:38472`

**验收**：Linux 双击打开应用窗口，退出后端口与 Solver 清理干净。

**本地构建前置**：`libwebkit2gtk-4.1-dev` 等 GTK 开发包（见 `apps/desktop/README.md`）。

**无 GTK 冒烟**：`bun run desktop:smoke`（仅验证 sidecar `TCH_DESKTOP=1` + `/health` + `/` HTML）。

---

### P5-E — 三平台 WebView（L）

**目标**：Linux / Windows / macOS 壳联调。

**验收**：各平台至少一种安装格式可运行（见 EN-08）。

---

### P5-F — 首次向导 + 托盘（M）

**目标**：API Key、VPS SSH、execSurface 向导；托盘菜单。

---

### EN-08 — 安装包流水线（L，随 P5-E）

**涉及**

- `scripts/pack-desktop.sh`
- GitHub Release 附件（可选）
- Linux AppImage/deb、Windows msi/nsis、macOS dmg

```
P5-A → P5-B → P5-C → P5-D → P5-E + EN-08 → P5-F
```
