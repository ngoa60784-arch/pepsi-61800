# BreachWeave 配置手册

> 逐项说明 **Web 配置页** 与 **`~/.tch-agent/config/`** 下各文件的格式、含义与推荐流程。  
> 项目背景与角色见 [项目文档.md](项目文档.md)。

---

## 1. 配置总览

| 存储根目录 | 默认路径 |
|----------|----------|
| 数据根 | `~/.tch-agent/` |
| 配置目录 | `~/.tch-agent/config/` |

**原则**

- 密钥、SSH 密码、FOFA、API Key **只存在本机 config**，不进 git。  
- 内置 Prompt/MCP/Skill 首次启动**种子化**；已有用户配置**不会被覆盖**。  
- 修改 `mcp.json` 或 Solver 相关 env 后，需 **重启或新建 Solver 容器**。

### 1.1 配置文件一览

| 文件 / 目录 | UI 对应 | 说明 |
|-------------|---------|------|
| `settings.json` | 配置 → **调度器** | 运行时、Planner、全局默认模型 |
| `auth.json` | 配置 → **API 密钥** | LLM 提供商密钥（SDK AuthStorage） |
| `provider-prefs.json` | 配置 → **提供商** | 多线路由配置（baseUrl、api 等） |
| `models.json` | （自动同步） | SDK ModelRegistry，与 provider-prefs 联动 |
| `model-prefs.json` | 配置 → **模型** | 用户选用的模型条目（id、thinking 等） |
| `mcp.json` | 配置 → **MCP** | MCP 服务定义与环境变量 |
| `mcp-cache.json` | （自动） | MCP 工具发现缓存 |
| `prompts/*.md` | 配置 → **提示词** | Agent/Subagent 系统提示 |
| `prompts/.deleted-builtins.json` | （自动） | 已删除的内置 Prompt 记录 |
| `skills/` | 配置 → **技能** | 用户 Skill 目录 |
| `tools`（注册表） | 配置 → **工具** | 内置 + 自定义工具元数据 |

---

## 2. 推荐配置顺序（首次）

1. **API 密钥**：至少一个 LLM 提供商。  
2. **提供商 / 模型**：注册模型并 **测试连通**；在调度器或模型页 **设为全局默认**。  
3. **MCP → kali-arsenal**：SSH（或别名）→ **测试 SSH** → 可选 FOFA → **同步 FOFA 到 Kali** → 可选一键装环境。  
4. **MCP → vuln-intel**（可选）：`NVD_API_KEY`、`GITHUB_TOKEN`。  
5. **调度器**：`maxSolvers`、内存/CPU、Planner tick。  
6. **环境变量**：`export TCH_ENGAGEMENT_SCOPE=/path/to/scope.json` 后重启 Web。  
7. **公网部署**：`TCH_AUTH_TOKEN` + systemd（见项目文档 §10）。

## 3. settings.json（调度器 / Host）

路径：`~/.tch-agent/config/settings.json`  
结构：顶层可含 SDK 其他字段；BreachWeave 使用 **`tch`** 对象：

```json
{
  "tch": {
    "defaultModelPrefId": "abc123",
    "runtime": {
      "image": "tch-agent:latest",
      "maxSolvers": 3,
      "memory": "4g",
      "cpus": 2,
      "networkMode": "host",
      "env": {},
      "solverEnv": {},
      "binds": []
    },
    "planner": {
      "tickIntervalMs": 30000,
      "staleTimeoutMs": 600000,
      "strategy": ""
    },
    "challenge": {}
  }
}
```

### 3.1 runtime（Solver 容器）

| 字段 | 说明 |
|------|------|
| `image` | Docker 镜像，默认 `tch-agent:latest`；可填私有仓库完整路径 |
| `maxSolvers` | 全局最大并发 Solver 数（Planner 遵守） |
| `memory` | 每容器内存上限，Docker 语法：`2g`、`512m`；空=不限 |
| `cpus` | 每容器 CPU 上限，如 `1.5`；空=不限 |
| `networkMode` | `host`（默认）或 `bridge` |
| `env` | 注入**宿主机侧** runtime 的环境变量 |
| `solverEnv` | 注入 **Solver 容器**的环境变量 |
| `binds` | 额外 Docker volume 绑定 |

### 3.2 planner

| 字段 | 说明 |
|------|------|
| `tickIntervalMs` | Planner 轮询间隔，默认 30000 |
| `staleTimeoutMs` | 判定 Solver/目标 stale 的超时 |
| `strategy` | 写入 Planner 上下文的运营策略文本（UI 可编辑） |
| `enabled` | 代码层恒为 true，不可通过 UI 关闭调度 |

修改后保存即写入文件；**Planner 逻辑在 Web 进程内**，改 `CHALLENGE_PLANNER` 或 tick 相关代码后需 **重启 `bun run web`**。

### 3.3 defaultModelPrefId

- 指向 `model-prefs.json` 中某条的 `id`。  
- UI **「设为全局默认」** 会：写入此项，并把未单独指定模型的 Prompt（含 Planner、Verifier）的 frontmatter `model` 同步为该 id。

## 4. API 密钥（auth.json）

- 由 SDK `AuthStorage` 管理，路径 `config/auth.json`。  
- UI **配置 → API 密钥** 按**提供商 id** 读写。  
- 格式为 SDK 内部结构（`type: api_key` 等），**勿手工编辑**，除非迁移备份。

---

## 5. 提供商与模型

### 5.1 provider-prefs.json

```json
{
  "providers": [
    {
      "id": "unique-id",
      "name": "显示名",
      "api": "openai-completions",
      "baseUrl": "https://api.example.com/v1",
      "apiKey": "sk-..."
    }
  ]
}
```

- 支持同一「显示名」多条线路（多 id）。  
- 保存时会同步相关条目到 `models.json`。

### 5.2 model-prefs.json

```json
{
  "models": [
    {
      "id": "pref-id",
      "provider": "显示名或逻辑名",
      "providerId": "provider-prefs 的 id",
      "modelId": "gpt-4o",
      "thinkingLevel": "high"
    }
  ]
}
```

- `id` 短字符串：Prompt frontmatter 里 `model:` 引用的是此 id。  
- **测试模型**：UI 调用 `/api/config/test-model`。  
- **激活全局**：`/api/config/activate-model`。

### 5.3 models.json

- SDK **ModelRegistry** 持久化；与 pref 同步。  
- 一般通过 UI 维护，避免手改导致注册表不一致。

## 6. MCP（mcp.json）

路径：`~/.tch-agent/config/mcp.json`

```json
{
  "mcpServers": {
    "kali-arsenal": {
      "command": "python3",
      "args": ["/opt/tch-mcp/ssh_mcp.py"],
      "env": { "SSH_HOST": "...", "FOFA_KEY": "..." },
      "lifecycle": "keep-alive"
    },
    "vuln-intel": {
      "command": "python3",
      "args": ["/opt/tch-mcp/vuln_intel_mcp.py"],
      "env": { "NVD_API_KEY": "", "GITHUB_TOKEN": "" },
      "lifecycle": "keep-alive"
    }
  },
  "settings": {
    "toolPrefix": "server",
    "idleTimeout": 10,
    "directTools": false
  }
}
```

### 6.1 kali-arsenal 环境变量

| 变量 | 必填 | 说明 |
|------|------|------|
| `SSH_ALIAS` | 二选一 | `~/.ssh/config` 别名（优先于 HOST/PASS） |
| `SSH_HOST` | 二选一 | 远程 IP/域名 |
| `SSH_PORT` | 否 | 默认 `22` |
| `SSH_USER` | 否 | 默认 `root` |
| `SSH_PASS` | 视情况 | 密码登录；别名+密钥可留空 |
| `FOFA_EMAIL` | 可选 | FOFA 账号邮箱 |
| `FOFA_KEY` | 可选 | FOFA API Key |
| `FOFA_EMAIL_2` | 可选 | 备用账号 |
| `FOFA_KEY_2` | 可选 | 备用 Key |
| `TCH_GOPROXY` | 可选 | 远程 Go 代理，如 `https://goproxy.cn,direct` |
| `TCH_GH_MIRROR` | 可选 | GitHub 镜像；**国外 VPS 勿填** |

UI 以键值对编辑，保存后写入 `mcp.json`。**不要**把含密钥的 `mcp.json` 提交到 git。

### 6.2 FOFA 同步到远程 Kali

1. 填写 `FOFA_EMAIL` + `FOFA_KEY`。  
2. **测试 SSH** 成功。  
3. 点击 **同步 FOFA 到 Kali**（API：`POST /api/config/mcp/kali-ssh-test`，body 含 `mode: "sync-keys"`）。  
4. 远程写入 `/root/.pentest-keys/keys.env`，并在 `.bashrc` 追加 source。  
5. **一键配置环境** 成功且已填 FOFA 时会尝试自动同步。

pentest skill 从该文件读取变量，与仓库内 `keys.env` 约定一致。

### 6.3 vuln-intel

| 变量 | 说明 |
|------|------|
| `NVD_API_KEY` | NVD API（可选，提高限额） |
| `GITHUB_TOKEN` | GitHub API（可选） |

控制面在 Solver `record_asset(service)` 时会自动查 NVD + CISA KEV（`packages/core` 侧），结果缓存于 `~/.tch-agent/cache/nvd.json`（TTL 24h），与 MCP 侧串行限速策略一致。限流或排队时会在目标 memory 写入说明，并在工具返回中带 `vuln_lookup` 状态。

### 6.4 mcp.settings

| 字段 | 说明 |
|------|------|
| `toolPrefix` | `server` / `none` / `short` — MCP 工具名前缀策略 |
| `idleTimeout` | 空闲断开（分钟），0=禁用 |
| `directTools` | 是否直连注册部分工具 |

### 6.5 路径迁移

- 旧版若使用仓库内 `mcp/xxx.py` 路径，启动时会自动改为 `/opt/tch-mcp/xxx.py`。  
- 宿主机 **探测 MCP** 时解析回仓库真实路径执行。

### 6.6 新增自定义 MCP

UI **添加服务器**：指定 `command`、`args`、`env`。  
Solver 容器需能执行该 command（或仅在宿主机 probe 测试）。

## 7. 提示词（prompts/*.md）

路径：`~/.tch-agent/config/prompts/<名称>.md`

### 7.1 文件格式

```markdown
---
description: 简短说明
model: model-pref-id
observerEnabled: true
observerModel: another-pref-id
mcps:
  - kali-arsenal
  - vuln-intel
tools:
  - bash
  - report_finding
skills:
  - pentest
subagents:
  - VULN_RESEARCHER
isSubagent: false
---

Markdown 正文 = 系统提示词
```

| frontmatter | 说明 |
|-------------|------|
| `model` | `model-prefs.json` 中的 `id`；空则用全局默认 |
| `observerEnabled` | 是否启用 Observer 旁路 |
| `observerModel` | Observer 专用模型 id |
| `mcps` | 白名单；**空数组=禁用全部 MCP** |
| `tools` | 工具白名单；含 skill 时通常自动加 `read` |
| `skills` | Skill 名称列表 |
| `subagents` | 允许调用的子代理 Prompt 名 |
| `isSubagent: true` | 仅作子代理，不出现在 Solver 启动列表 |

### 7.2 内置 Prompt

首次启动从包内释放到 `prompts/`（若不存在）：

| 名称 | 用途 |
|------|------|
| `CHALLENGE_PLANNER` | 调度器 |
| `OBJECTIVE_VERIFIER` | 验证者 |
| `KALI_PROVISIONER` | Kali 装工具 |
| `kimi-security` | 主渗透 Solver |
| `VULN_RESEARCHER` | 子代理 |

删除内置会在 `.deleted-builtins.json` 记录，避免再次释放。

## 8. 技能（skills/）

- 目录：`~/.tch-agent/config/skills/<skill-name>/`，含 `SKILL.md` 等。  
- UI 可上传 zip、从 git 导入、删除。  
- 内置 skill 在包内 `packages/core/src/config/skills/builtin/`，启动时复制缺失项到用户目录。  
- Prompt frontmatter `skills:` 列出启用名。

---

## 9. 工具（tools）

- 内置：bash、read、engagement 相关（`report_finding`、`record_asset` 等）。  
- UI **配置 → 工具** 注册自定义工具定义。  
- Prompt `tools:` 白名单控制 Solver 可用集合。

## 10. Web 认证

| 变量 | 行为 |
|------|------|
| `TCH_AUTH_TOKEN` | 非空：所有 `/api/*` 与 SSE 需登录；Cookie `tch_auth` 或 `Authorization: Bearer` |

未设置时 **无任何认证**，仅限本机或可信网络。

生成示例：

```bash
openssl rand -hex 32
export TCH_AUTH_TOKEN=<输出>
bun run web
```

## 11. 环境变量完整表

| 变量 | 作用域 | 说明 |
|------|--------|------|
| `TCH_AUTH_TOKEN` | Web | 启用登录 |
| `TCH_ENGAGEMENT_SCOPE` | 进程 | 作战范围 JSON 绝对路径 |
| `TCH_ENGAGEMENT_MODE` | 进程 | `0` 仅测试用；生产默认 engagement |
| `TCH_AGENT_SKIP_PYTHON_DEPS` | install | 跳过 MCP Python 依赖 |
| `TCH_AGENT_SKIP_INSTALL_SCRIPT` | install | 跳过 postinstall 脚本 |
| `TCH_CHALLENGE_DIR` | 测试 | 覆盖 challenge 目录 |
| `TCH_SOLVER_SESSION_DIR` | 容器内 | Solver 会话目录 |
| `TCH_SOLVER_WORKSPACE` | 容器内 | 工作区路径 |

MCP `kali-arsenal` 的 `env` 见 §6.1（存在 `mcp.json`，非系统环境变量）。

## 12. 作战范围（Engagement Scope）

**不**写在 `config/` 内，用环境变量指向独立 JSON：

```bash
export TCH_ENGAGEMENT_SCOPE=/etc/tch/engagement-scope.json
```

模板：[engagement-scope.example.json](engagement-scope.example.json)

| 字段 | 必填 | 说明 |
|------|------|------|
| `engagement` | 是 | 演练名称 |
| `allowed_targets` | 是（非空） | IP / 域名 / CIDR / URL 前缀白名单 |
| `out_of_scope` | 否 | 排除项，优先于白名单 |
| `no_scan` | 否 | true 时禁用 nmap/ffuf 等主动扫描语义（软约束） |
| `forbidden_commands` | 否 | 额外禁止命令 token |
| `rules_of_engagement` | 否 | 自由文本 ROE |

未设置或 `allowed_targets` 为空时，**不会**向 Solver 注入范围文本（强烈不推荐生产使用）。

## 13. 安全与备份

| 项 | 建议 |
|----|------|
| 备份 | `config/` + `challenge/`；`solvers/` 可重建 |
| 权限 | `chmod 700 ~/.tch-agent`；`mcp.json` 含 SSH/FOFA 密码 |
| git | `.gitignore` 应忽略 `keys.env`；勿提交 `mcp.json` 副本 |
| 范围文件 | 含真实目标信息，勿入仓 |
| 凭据证据 | findings 用 `evidence_refs`，避免明文进共享 memory |

## 15. 故障排查（配置向）

| 现象 | 检查 |
|------|------|
| 模型测试失败 | `auth.json`、provider `baseUrl`/`api`、网络 |
| Solver 无 MCP 工具 | Prompt `mcps` 是否列出；容器是否重启 |
| SSH 测试失败 | `sshpass`（密码模式）、别名、防火墙 |
| FOFA 同步 404 | 重启 Web；确认请求带 `mode: sync-keys` |
| Planner 用错模型 | `defaultModelPrefId` 与 Prompt `model` |
| 改 settings 不生效 | runtime 字段需新 Solver；planner tick 需重启 Web |
